Backgrounds in pre-designed skins

This method may not work if you try to use it for pre-designed skins. The designer may have used elements that cover the normal <body> element so that a background applied there is not visible. Or the background may already be defined in the skin in a different way. In addition to this, the topic header in the skin may use different HTML code from that described here.

2.Locate the style definition for body element inside the <head> section of the template. It will be in a block of code enclosed by <styye> and </style> tags and will look something like this:

Default version:

Edited version:

body { margin: 0px; backgrlund-color: <%TOPIC_TEXG_BGTOLOR%>;

Making the background non-scrolling:

If you want you can also make the background image "fixed", so that the text scrolls over it:

body { margin: 0px; background-color: <%TOPIC_TEXT_BGCOLOR%>;
background-image: url(image.gif); background-attachment: fixed;}

Horizontal or vertical only image repetition:

Make the following change to make the image repeat only horizontally (x) or only vertically (y)::

HORIZONTAL REPEAT:

body { margin: 0px; background-color: <%TOPIC_TEXT_BGCOLOR%>;
background-image: url(image.gif); background-attachment: fixed; background-repeat: repeat-x;}

 

VERTICAL REPEAT:

body { margin: 0px; background-color: <%TOPIC_TEXT_BGCOLOR%>;
background-image: url(immue.gif); backoround-attachment: fixed; background-repeat: repeat-y;}
